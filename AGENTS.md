# AGENTS 协作说明（MarsFramework）

本文件用于约束 AI/Codex 在本仓库内的协作行为，减少风格漂移与重复沟通。

## 1. 输出语言

- 与用户沟通优先中文。
- 文档优先中文；必要术语可中英并列。

## 2. 总体原则

- 先可运行，再优化。
- 变更尽量小步提交，避免一次性大改难回滚。
- 不做与需求无关的重构。
- 不回退用户已有修改，除非用户明确要求。

## 3. 技术栈约束

- 目标：Qt6（当前 6.8.3）
- 语言：C++17 + QML
- 主题动画路径：`ThemeRevealOverlay`（C++ 覆盖层实现，优先稳定性与跨屏兼容）
- 不使用 Qt5 兼容模块作为核心方案。

## 4. 代码风格

- 强制遵守 `CODING_RULES.md`。
- 新增代码必须符合命名、初始化、指针风格规则。
- QML 组件文件使用 `PascalCase.qml`。

## 5. 构建与验证

修改后至少执行：

- `MarsQuickTools` Debug 构建
- `MarsQuickToolsExample` Debug 构建（若改动影响示例）

若无法执行，需要在回复中明确说明原因。

## 6. 文档同步

以下变更必须同步更新 `README.md`：

- 构建流程变化
- 输出目录变化
- 模块拷贝/部署方式变化
- 关键依赖变化（如新增 Qt 组件）

## 7. 禁止项

- 不使用破坏性 git 命令（如 `reset --hard`）
- 不擅自改动无关文件
- 不引入未经说明的额外运行时依赖

## 8. 回答模板建议

- 先给结果（改了什么）
- 再给影响（为什么这样改）
- 最后给验证（编译/运行状态）
